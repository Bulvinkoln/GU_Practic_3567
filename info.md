# Что такое Git? 

*Git* - это консольная утилита, для отслеживания и ведения истории изменения файлов, в вашем проекте. Чаще всего его используют для кода, но можно и для других файлов. Например, для картинок - полезно для дизайнеров.

## Непонятные слова:

**Репозиторий** - это хранилище вашего кода и историю его изменений. Git работает локально и все ваши репозитории хранятся в определенных папках на жестком диске. Если уже совсем на простом языке то это папка!

**Коммит (commit)** - точка сохранения вашего проекта. У каждого commit-a есть hash (уникальный id) и комментарий. Из таких commit-ов собирается ветка. Ветка - это история изменений. У каждой ветки есть свое название. Репозиторий может содержать в себе несколько веток, которые создаются из других веток или вливаются в них.

## Как работает Git? 

Если посмотреть на картинку, то становиться чуть проще с пониманием. Каждый кружок, это commit. Стрелочки показывают направление, из какого commit сделан следующий. Например C3 сделан из С2 и т. д. Все эти commit находятся в ветке под названием main. Это основная ветка, чаще всего ее называют master . Прямоугольник main* показывает в каком commit мы сейчас находимся, проще говоря указатель.

## Как установить?

**Windows** проходим по этой [ссылке](https://git-scm.com/download/win), выбираем под вашу ОС (32 или 64 битную), скачиваем и устанавливаем.

**Mac OS** открываем терминал и пишем: 

**Если установлен Homebrew**:

brew install git

**Если нет,** то вводим эту команду. 

git --version

После этого появится окно, где предложит установить **Command Line Tools (CLT).** Соглашаемся и ждем установки. Вместе с CLT установиться и git

**Linux.** Открываем терминал и вводим следующую команду:

**Debian или Ubuntu**

*sudo apt install git*

**CentOS**

*sudo yum install git*

## Коммитим (commit) 

Как мы уже знаем поработая с документом проектом и тд. тп. его нужно сохранять для этого есть команда 

**git commit -m "nametrack"**

где -m "nametrack"  - это сообщение, которое поможет опредеелить отрезок либо еще какую важную вещь как другому человеку так и самому себе.

## Создание ветки

Для создания ветки нам необходимо изначально "закомитить" основную нашу ветку у меня это (main) у Вас может быть (master)

После проделанной работы нам понадобятся команды: 

**git branch** - показывает список веток которые у нас есть впринципе в данном проекте.

**git branch** _name_vashei_vetki_ - данная команда создает ветку с названием которое вы сами придумали обратите внимание, что когда вы используете 2 слова в имени то оно пишется без пробелов а с нижним прочерком. 

## Переход между ветками

Для перехода между ветками намбудет необходима команда: 

**git checkout** *nazvanie_vetki* - аналогично гит чекаут это команда название ветки это название ветки на которую вы хотите перейти

Далее снова используем нашу команду **git branch** для того чтобы посмотреть на какой ветке мы находимся
 
## Мерж веток (слияние)

Для слияния веток необходимо понять какая же у нас ветка главная. Как я уже говорил у меня это ветка main у вас скорее всего master. Так вот для мержа (от англ слияние) нам необходимо выполнить след алгоритм: 

**git branch** - узнаем на какой мы сейчас ветке.

**git checkout nazvanie_vetki** - для перехода на основную ветку (либо же на ветку с которой мы хотим смержить)

**git merge nazvanie_vetki**  - если у вас нет конфликтов в ветке то у вас успешно пройдет мерж основной ветки и дополнительной которую вы создавали.

## Конфликты

**Что такое конфликты?**

Бывает так что в разных ветках один и тот же текст файл тип и тд и тп редактируется и когда эти ветки будут сливаться возникает конфликт и git не понимает какую версию необходимо оставить, изначальную версию либо версию из ветки либо оба варианта. 

Если у вас возник конфликт необходимо выбрать в самой программе вариант либо отредактировать самому слив 2 варианта.
